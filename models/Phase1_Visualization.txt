# Phase 1 추가: MLP 모델 분석 및 시각화 가이드 (Pandas 활용)

이 단계에서는 이전에 학습하여 저장한 모델 파일(예: `mlp_model_v1.h5`)을 불러와서 
모델의 예측 결과, 가중치 분포, 학습 지표 등을 `pandas` 데이터프레임을 활용하여 구조적으로 분석하고 시각화하는 과정(예: `visualize_model.py` 또는 주피터 노트북)을 설명합니다.

## 단계 1: 분석 및 시각화를 위한 라이브러리 설치
Pandas 기반의 데이터 분석과 시각화를 위해 필요한 라이브러리들을 설치합니다.
- 설치 명령어: `pip install pandas matplotlib pydot graphviz`
  *(주의: `graphviz`의 경우 파이썬 패키지 외에도 OS 시스템 레벨의 프로그램 설치가 추가로 필요할 수 있습니다. 설치가 번거롭다면 다이어그램 추출은 생략해도 무방합니다.)*

## 단계 2: 저장된 모델 불러오기 및 레이어 정보 분석
1. Keras/TensorFlow를 사용해 앞서 학습 후 저장한 모델 파일을 로드합니다.
   - 예: `model = keras.models.load_model('mlp_model_v1.h5')`
2. **Pandas를 활용한 구조 파악**: 모델의 레이어 정보를 추출하여 DataFrame으로 변환해 표 형태로 직관적으로 확인합니다.
   - 레이어의 이름(Name), 타입(Type), 출력 형태(Output Shape), 파라미터 개수(Params)를 딕셔너리로 모아 `pd.DataFrame()`에 넣습니다.

## 단계 3: 예측 결과의 데이터프레임화 (Pandas DataFrame)
테스트 데이터의 예측 결과를 DataFrame으로 만들어 정답과 오답을 쉽게 필터링하고 분석합니다.
1. MNIST 테스트 데이터를 로드하고 스케일링(1/255.0) 처리를 합니다.
2. 모델을 통해 테스트 데이터 전체에 대한 예측값(예측된 최종 클래스 숫자)을 구합니다.
3. **결과 결합**: 예측된 레이블(Predicted)과 실제 정답 레이블(Actual)을 엮어서 DataFrame을 생성합니다.
   - 예: `df = pd.DataFrame({'Actual': y_test, 'Predicted': y_pred_classes})`
4. 정답 여부를 나타내는 컬럼(`Correct`)을 추가합니다 (`df['Actual'] == df['Predicted']`).

## 단계 4: Pandas를 이용한 분석 및 시각화
Pandas 객체에 내장된 `.plot()` 기능과 기초 통계 기능을 활용합니다.
1. **오답 빈도 시각화**: `df[~df['Correct']]`로 오답만 필터링한 뒤, 실제 정답이었던 숫자가 무엇일 때 가장 많이 틀렸는지 집계하여 막대 그래프(`bar chart`)로 시각화합니다.
   - 예: `error_df['Actual'].value_counts().sort_index().plot(kind='bar')`
2. **오답 매트릭스(Cross Tabulation)**: `pd.crosstab(df['Actual'], df['Predicted'])`을 사용하여 Confusion Matrix(오차 행렬)를 생성하고 텍스트 테이블 형태로 모델의 취약점을 파악합니다.
3. **가중치 분석**: 특정 은닉층의 가중치를 1차원 배열로 펼친 후, DataFrame에 담아 기초 통계량(`describe()`)을 확인하고 히스토그램(`plot(kind='hist')`)으로 분포를 그립니다.

## 단계 5: 특정 오답 케이스 이미지 확인 (Matplotlib 결합)
Pandas로 필터링해낸 오답 데이터의 인덱스(Index)를 활용하여, 실제 틀린 이미지만 선별적으로 시각화합니다.
1. 앞서 생성한 DataFrame에서, 예를 들어 실제로는 3인데 8로 예측한 데이터(`Actual == 3 & Predicted == 8`)의 인덱스 번호들을 추출합니다.
2. 해당 인덱스들을 이용해 실제 28x28 이미지 배열을 가져오고, `matplotlib.pyplot.imshow()`를 통해 화면에 띄워 왜 모델이 헷갈렸는지 사람의 눈으로 확인합니다.

위 과정을 바탕으로 `models` 폴더 내에 `visualize_model.py` (또는 주피터 노트북) 파일을 만들어 코드를 작성하시면 됩니다.
